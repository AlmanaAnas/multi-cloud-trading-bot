Built with the right tools

Every technology was chosen for a specific reason — not just for the project, but to demonstrate production infrastructure thinking.

🏗️
Terraform IaC
Reusable modules for AWS and GCP. Remote state in S3 with DynamoDB locking. Environment isolation between dev and prod. Every resource declared as code.
modules/aws · modules/gcp · modules/shared
🔐
Zero credentials stored
GitHub Actions authenticates to both clouds via OIDC. Lambda authenticates to GCP Pub/Sub via Workload Identity Federation. No access keys. No service account files.
OIDC + Workload Identity Federation
⚡
Serverless compute
AWS Lambda (Python 3.11) for data ingestion. GCP Cloud Function 2nd gen for ML inference. Both scale to zero when idle — zero cost outside active runs.
Lambda
Cloud Functions
📨
Event-driven messaging
GCP Pub/Sub with push subscription, dead-letter topic, and exponential backoff retry. Fan-out topology allows new consumers to subscribe without touching the producer.
Pub/Sub · DLQ · push sub
🚀
GitHub Actions CI/CD
terraform plan on every PR. terraform apply on merge to main. Manual destroy workflow for one-click teardown. Infrastructure as code deployed as code.
plan · apply · destroy
📊
BigQuery analytics
All trading signals archived to a day-partitioned BigQuery table. Schema enforced at the Terraform level. Ready for model retraining and Looker Studio dashboards.
day partitioned · streaming insert

Security

Zero credentials stored anywhere

Every authentication boundary in the system uses short-lived tokens. No access keys. No service account JSON files. No secrets in environment variables.

🔑
GitHub → AWS
GitHub Actions assumes an IAM role via OIDC. The role trust policy is scoped to this exact repository — no other repo can assume it.
✓ No AWS_ACCESS_KEY_ID stored
🌉
GitHub → GCP
Workload Identity Federation maps GitHub's OIDC token to a GCP service account. The provider is locked to this repository by attribute condition.
✓ No service account key file
⚡
Lambda → GCP Pub/Sub
Lambda exchanges its AWS identity for a short-lived GCP token via Workload Identity Federation at runtime. The token expires in 1 hour.
✓ Cross-cloud keyless auth
🛡️
Least privilege IAM
Every role has exactly the permissions it needs — nothing more. Lambda can only publish to Pub/Sub. The GitHub role can only manage the specific services used.
✓ No AdministratorAccess on runtime roles
🗄️
State security
Terraform state is encrypted at rest in S3 (AES-256). Public access blocked at the bucket level. DynamoDB prevents concurrent state corruption.
✓ Encrypted · locked · private
💣
Clean teardown
A manual workflow_dispatch GitHub Action destroys all 41 resources in both clouds with one click. No lingering resources, no forgotten buckets, no ongoing costs.
✓ terraform destroy on demand
